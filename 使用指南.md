## 🎯 运行实验

### 方式 1：交互式菜单（推荐新手）

```bash
./start_experiment.sh
```

**菜单选项**：
1. 🎯 完整实验流程（VAE → Diffusion → FM → 评估）
2. ⏩ 跳过 VAE 预训练
3. 📊 仅运行对比评估
4. ⚙️ 自定义参数运行
5. 📖 查看帮助

### 方式 2：一键完整实验

```bash
# 完整流程（36-74 小时，RTX 4090）
./run_full_comparison_experiment.sh

# 查看所有可用参数
./run_full_comparison_experiment.sh --help
```

### 方式 3：分步执行

```bash
# 步骤 1：VAE 预训练 (180,000 步，12-24h)
python3 tools/train_generator.py \
  configs/generation/fpc/fpc_1a_latentc3_z4_pc64_180k.py

# 步骤 2：Diffusion 训练 (180,000 步，12-24h)
python3 tools/train_generator.py \
  configs/comparison/exp_diffusion_vs_fm.py \
  --model diffusion

# 步骤 3：Flow Matching 训练 (180,000 步，12-24h)
python3 tools/train_generator.py \
  configs/comparison/exp_diffusion_vs_fm.py \
  --model flow_matching

# 步骤 4：对比评估 (1-2h)
python3 tools/inference.py \
  configs/comparison/exp_diffusion_vs_fm.py \
  --compare
```

### 常用参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--skip-vae` | 跳过 VAE 预训练 | `./run_full_comparison_experiment.sh --skip-vae` |
| `--skip-diffusion` | 跳过 Diffusion 训练 | `./run_full_comparison_experiment.sh --skip-diffusion` |
| `--skip-fm` | 跳过 Flow Matching 训练 | `./run_full_comparison_experiment.sh --skip-fm` |
| `--skip-eval` | 跳过对比评估 | `./run_full_comparison_experiment.sh --skip-eval` |
| `--num-gpus N` | 指定 GPU 数量 | `./run_full_comparison_experiment.sh --num-gpus 2` |
| `--batch-size N` | 指定批大小 | `./run_full_comparison_experiment.sh --batch-size 64` |
| `--debug` | 启用调试模式 | `./run_full_comparison_experiment.sh --debug` |

### 使用多 GPU 加速

```bash
# 使用 2 个 GPU，批大小 64
./run_full_comparison_experiment.sh --num-gpus 2 --batch-size 64

# 指定特定 GPU
CUDA_VISIBLE_DEVICES=0,1 ./run_full_comparison_experiment.sh --num-gpus 2
```

## 📊 查看结果

### 方式 1：一键查看菜单（推荐）

```bash
./view_all_results.sh
```

**菜单功能**（16 个选项）：

- 📋 查看 CSV 对比表格
- 🎨 生成 PNG 可视化 HTML
- 📦 打包结果下载
- 📁 查看目录结构
- 📝 查看训练日志

### 方式 2：查看训练曲线（TensorBoard）

```bash
# 查看所有模型的训练曲线
tensorboard --logdir=./output/comparison/exp_diffusion_vs_fm --port=6006
# 浏览器打开：http://localhost:6006

# 仅查看 VAE
tensorboard --logdir=./output/comparison/exp_diffusion_vs_fm/vae/logs --port=6006

# 仅查看 Diffusion
tensorboard --logdir=./output/comparison/exp_diffusion_vs_fm/ddm/logs --port=6007

tensorboard --logdir=./output/comparison/exp_diffusion_vs_fm/fm/logs --port=6008
```

**在 WebIDE 中使用端口转发**：
- VS Code：`Ctrl+Shift+P` → `Remote: Forward a Port` → 输入 `6006`
- 或使用 SSH 本地转发：`ssh -L 6006:localhost:6006 user@host`

### 方式 3：查看对比数据（CSV 表格）

```bash

python3 view_csv_results.py

cat ./output/comparison/exp_diffusion_vs_fm/comparison_results/comparison_table.csv | column -t -s,
```



## ❓ 常见问题

### Q1: 如何检查实验是否在正常运行？

```bash
# 方法 1：查看日志
tail -f ./output/logs/full_experiment_*.log

# 方法 2：查看 GPU 使用情况
watch -n 1 nvidia-smi

# 方法 3：查看 TensorBoard
tensorboard --logdir=./output/comparison/exp_diffusion_vs_fm --port=6006
```

### Q2: 训练中断了怎么办？

graspLDM 支持自动从检查点恢复训练：

```bash
# 继续训练（会自动加载最新检查点）
./run_full_comparison_experiment.sh

# 或跳过已完成的步骤
./run_full_comparison_experiment.sh --skip-vae  # 如果 VAE 已完成
```

### Q3: 如何使用更少的显存？

```bash
# 减小批大小
./run_full_comparison_experiment.sh --batch-size 16

# 或在配置文件中修改
# configs/comparison/exp_diffusion_vs_fm.py
# 修改 batch_size 参数
```

### Q4: TensorBoard 无法连接？

```bash
# 检查 TensorBoard 是否运行
ps aux | grep tensorboard

# 重启 TensorBoard
pkill tensorboard
tensorboard --logdir=./output/comparison/exp_diffusion_vs_fm --port=6006

# 在 WebIDE 中，使用端口转发
# VS Code: Ctrl+Shift+P → Remote: Forward a Port → 6006
```

### Q5: 数据集在哪里？

ACRONYM 数据集位于：
```
data/ACRONYM/grasps/*.h5  (8837 个文件)
```

如果缺失，请参考 [docs/DEPLOYMENT_GUIDE_CN.md](docs/DEPLOYMENT_GUIDE_CN.md) 下载数据。

### Q6: 如何只运行特定模型？

```bash
# 只训练 VAE
python3 tools/train_generator.py configs/generation/fpc/fpc_1a_latentc3_z4_pc64_180k.py

# 只训练 Diffusion

# 只训练 Flow Matching
python3 tools/train_generator.py configs/comparison/exp_diffusion_vs_fm.py --model flow_matching
```

### Q7: 如何查看某个模型的详细日志？

```bash
# 查看 VAE 日志
tail -100 ./output/logs/01_vae_training_*.log

# 查看 Diffusion 日志
tail -100 ./output/logs/02_diffusion_training_*.log

tail -100 ./output/logs/03_flow_matching_training_*.log
```

### Q8: 完整实验需要多长时间？

| 步骤 | 训练步数 | GPU (RTX 4090) | GPU (RTX 3090) |
|------|---------|----------------|----------------|
| VAE 预训练 | 180,000 | 12-18 小时 | 18-24 小时 |
| Diffusion 训练 | 180,000 | 12-18 小时 | 18-24 小时 |
| Flow Matching 训练 | 180,000 | 12-18 小时 | 18-24 小时 |

## 💡 最佳实践

### 训练建议

1. **使用多 GPU**：如果有多个 GPU，使用 `--num-gpus` 参数加速
2. **调整批大小**：根据 GPU 显存调整 `--batch-size`
3. **定期检查点**：确保训练日志正常写入，检查点正常保存
4. **监控 GPU**：使用 `watch -n 1 nvidia-smi` 监控 GPU 使用情况
5. **使用 TensorBoard**：实时查看训练曲线，及早发现问题

### 结果查看建议

1. **使用一键菜单**：`./view_all_results.sh` 最方便
2. **TensorBoard 对比**：同时启动多个 TensorBoard 实例对比模型
3. **保存结果**：定期运行 `./package_results.sh` 备份结果
4. **生成报告**：使用 HTML 查看器生成可视化报告

### 故障排除建议

1. **训练中断**：graspLDM 会自动从检查点恢复，直接重新运行即可
2. **显存不足**：减小批大小或使用梯度累积
3. **日志太多**：定期清理 `./output/logs/` 目录
4. **端口占用**：使用不同端口或 `pkill tensorboard`
