# GraspLDM VAE 训练实验指南：问题排查、常用命令与后续流程

本文档针对 GraspLDM 项目中 VAE（变分自编码器）的训练实验，整理了**常见问题与解决方案**、**核心常用命令**以及**VAE 训练完成后的后续流程**，帮助你高效完成实验。

---

## 1. 实验概述
### 1.1 实验目标
训练一个 GraspCVAE 模型，用于抓取数据的隐空间编码与重构，为后续 Diffusion/Flow Matching 生成模型提供预训练的 VAE  backbone。
### 1.2 核心配置
- 目标步数：`max_steps = 180000`
- Batch Size：`batch_size = 32`
- 单 GPU 训练：`num_gpus = 1`
- 数据加载线程：`num_workers_per_gpu = 0`（笔记本适配，避免卡死）

---

## 2. 常见问题与解决方案
### 2.1 假训练：`global_step` 不涨，Loss 飙升
#### 现象
- Epoch 快速跳到 200+，但 `global_step` 始终卡在 1000 步；
- Loss 从 0.1 飙升到 0.75+；
- 步速异常快（> 7 it/s），无反向传播。
#### 原因
旧 checkpoint 中 `global_step` 被错误标记为“已完成”，训练器仅循环加载数据、不更新权重。
#### 解决方案
1. 终止所有训练进程：`pkill -f train_generator.py`
2. 清空旧 checkpoint：`rm -rf output/comparison/exp_diffusion_vs_fm/vae/*`
3. 配置文件设 `resume_training_from_last = False`，从头开始训练。

---

### 2.2 数据加载卡死：启动后无进度，无日志
#### 现象
- 训练启动后停在 `Sanity Checking` 或 `Epoch 0: 0%`；
- 无日志生成，GPU 利用率为 0%。
#### 原因
`num_workers_per_gpu` 太大（如 7），超出笔记本磁盘 IO 能力，数据加载线程挂起。
#### 解决方案
配置文件设 `num_workers_per_gpu = 0`，用主线程加载数据，避免线程卡死。

---

### 2.3 训练中断：如何从 checkpoint 继续
#### 现象
不小心按了 `Ctrl+C` 或终端关闭，训练中断。
#### 解决方案
1. 确认进程已终止：`ps -ef | grep train_generator.py | grep -v grep`（有输出则 `pkill -f train_generator.py`）
2. 配置文件设 `resume_training_from_last = True`
3. 重启训练：`python tools/train_generator.py --config configs/comparison/exp_diffusion_vs_fm.py --model vae --num-gpus 1 --batch-size 32`
4. 验证恢复：看到 `Restored all states from the checkpoint file` 即成功。

---

### 2.4 Loss 异常：第一轮 0.1，后续涨到 0.28
#### 现象
- 第一轮 Loss 低至 0.1，后续 Epoch 涨到 0.28 左右；
- Loss 缓慢波动，无骤涨。
#### 原因
- 第一轮 KL 散度预热未生效，仅计算重构损失；
- 后续 KL 散度权重拉满，重构损失 + KL 损失叠加，Loss 回到合理区间。
#### 解决方案
**无需干预**，这是 VAE 训练的正常流程，训练后期（5 万步后）Loss 会波动中缓慢下降。

---

### 2.5 日志/Checkpoint 相关问题
#### 问题1：找不到 `train.log` 文件
- 原因：训练框架未将输出写入 `.log` 文件，仅输出到 tmux 终端。
- 解决：直接连接 tmux 会话看实时输出：`tmux attach -t vae_train_resume`

#### 问题2：`torch.load()` 加载 checkpoint 慢/中断
- 原因：默认加载到 GPU，显存占用大，加载慢。
- 解决：强制加载到 CPU：
  ```python
  import torch
  ckpt = torch.load("path/to/last.ckpt", map_location="cpu")
  print(f"当前步数: {ckpt['global_step']}")
  ```

## 4. VAE 训练完成后做什么
### 4.1 验证 VAE 重构效果
训练完成后，加载 `best.ckpt` 或 `last.ckpt`，验证模型对抓取数据的重构能力：
```python
import torch
from models.grasp_cvae import GraspCVAE

# 加载模型
ckpt = torch.load("output/comparison/exp_diffusion_vs_fm/vae/checkpoints/best.ckpt", map_location="cpu")
model = GraspCVAE(**ckpt["hyper_parameters"])
model.load_state_dict(ckpt["state_dict"])
model.eval()

# 验证重构（替换为你的验证数据）
sample_data = torch.randn(1, 3, 224, 224)  # 示例输入
with torch.no_grad():
    recon_data, z_mean, z_logvar = model(sample_data)
    print(f"重构误差: {torch.nn.functional.mse_loss(recon_data, sample_data):.4f}")
```

---

### 4.2 生成隐空间样本
利用训练好的 VAE，从隐空间采样生成新的抓取数据：
```python
import torch

# 从标准正态分布采样隐向量
z = torch.randn(10, model.latent_dim)  # 生成 10 个样本

# 解码生成抓取数据
with torch.no_grad():
    generated_data = model.decode(z)
    print(f"生成数据形状: {generated_data.shape}")

# 保存生成结果（根据你的数据格式调整）
torch.save(generated_data, "output/comparison/exp_diffusion_vs_fm/vae/generated_samples.pt")
```

### 4.3 后续实验：训练 Diffusion/Flow Matching 模型
VAE 训练完成后，将其作为预训练 backbone，训练 Diffusion 或 Flow Matching 生成模型：
1. **修改配置文件**：
   ```python
   # 在 configs/comparison/exp_diffusion_vs_fm.py 中设置
   vae_ckpt_path = "output/comparison/exp_diffusion_vs_fm/vae/checkpoints/best.ckpt"
   model = "ddm"  # 或 "fm" 用于 Flow Matching
   max_steps = 200000  # Diffusion 训练步数
   ```
2. **启动 Diffusion 训练**：
   ```bash
   python tools/train_generator.py \
     --config configs/comparison/exp_diffusion_vs_fm.py \
     --model ddm \
     --num-gpus 1 \
     --batch-size 32
   ```

## 5. 附录：配置文件关键参数
| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `resume_training_from_last` | 是否从最新 checkpoint 继续训练 | 中断后设 `True`，从头跑设 `False` |
| `max_steps` | 总训练步数 | VAE: 180000; Diffusion: 200000 |
| `batch_size` | 批次大小 | 单 GPU（12GB）: 32 |
| `num_gpus` | GPU 数量 | 1 |
| `num_workers_per_gpu` | 数据加载线程数 | 笔记本设 `0`，服务器设 `4~8` |
| `vae_ckpt_path` | 预训练 VAE 权重路径 | Diffusion 训练时设为 VAE 的 `best.ckpt` |

---

## 6. 总结
- **核心原则**：优先看 `global_step` 是否累加、GPU 利用率是否正常，这是判断训练有效的核心标准；
- **常见问题**：假训练、数据加载卡死、训练中断，按本文档步骤即可解决；
- **后续流程**：VAE 训练完成后，先验证重构效果，再生成样本，最后训练 Diffusion/Flow Matching 模型。

如果实验中遇到新问题，可结合 tmux 实时输出和本文档排查，或参考项目官方文档。
